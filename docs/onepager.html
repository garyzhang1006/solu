💧 Solu — See why molecules dissolve

An in-browser AI chemistry tutor that predicts a molecule's water solubility and teaches the structure–property reasoning behind it.

DSH Hacks V1 · Theme: AI × STEM Education · Web app (no install) · github.com/garyzhang1006/solu

The problem

Chemistry students memorize the rule "like dissolves like" but rarely see why one molecule dissolves and another doesn't. Solubility depends on a tug-of-war between polarity, hydrogen bonding, and lipophilicity that stays invisible in a textbook. Meanwhile, most "AI for education" tools are thin wrappers around a chat model — non-deterministic, unverifiable, and disconnected from real scientific data.

What Solu does

  • Type a SMILES string or pick from 14 example molecules.
  • RDKit-JS draws the 2D structure and computes 9 descriptors live, in the browser.
  • A trained gradient-boosting model predicts log-solubility (mol/L), converted to mg/L and a qualitative class.
  • A "Why?" panel decomposes the prediction into per-property contributions (LogP, polar surface area, H-bond donors…) a student can learn from.
  • Quiz mode for active recall; built-in lessons on polarity and hydrogen bonding.

The data & the model

Trained on the ESOL / Delaney dataset — 1,128 compounds with measured aqueous solubility, published by Delaney (2004) and distributed via MoleculeNet (Wu et al., Chem. Sci. 2018, Stanford). Features are the interpretable Delaney descriptor family computed with RDKit.

Why it's technically credible

The model trains in Python (RDKit + scikit-learn) but runs in the browser (RDKit-JS + ONNX Runtime Web). We discovered and fixed a real train/inference skew (RDKit changed its H-bond-acceptor definition between versions), then locked correctness with automated parity gates: RDKit-JS vs RDKit-Python descriptors agree to 1e-5, and the full browser inference chain reproduces the Python prediction bit-for-bit (diff = 0.0). Browser predictions are provably identical to the trained model.

Tech stack

RDKit & RDKit-JS · scikit-learn · skl2onnx + ONNX Runtime Web · vanilla HTML/CSS/JS (no framework, no backend).
